# 運営者向けマニュアル

展覧会の作成からキャプション印刷、データ書き出しまで。

---

## 0. 用語と画面の流れ

- **展覧会コード `ex_code`** (例: `TE001`): 各展覧会のユニーク ID
- **管理者ログイン (メールリンク認証)**: 「作品登録/設定」画面・「キャプション」画面・「データ書き出し」画面を開く際に必要。メールアドレスを入力 → 届いたリンクをクリックでログイン。一度ログインすると同じブラウザでは数日間維持される

> 作家用の入力画面 (作品入力) はログイン不要・URL 秘匿運用のままです。
> 来場者用の感想ページもログイン不要です。

主要画面と用途：

| 画面 | 用途 |
| --- | --- |
| 展覧会作成 | 新規展覧会の登録 (運営者のスタート地点) |
| 作品登録/設定 | 項目設定・案内メール・作品枠追加・感想受付期間・**作品ロック** |
| 作品入力 | 作家が作品データを入力 (招待 URL 経由) |
| キャプション | キャプション印刷・QR 印刷・テンプレート管理 |
| ダッシュボード | いいね/コメントのリアルタイム表示 (会場 + Web 別ランキング) |
| データ書き出し | 作品データ・コメント・いいねの XLSX / CSV 取得 |
| **Web 展覧会設定** | gallery.html の公開設定・外観・会場 QR 発行 |

よく使う画面はブラウザのブックマークに登録しておくのがおすすめです。

---

## 1. 新しい展覧会を作る

1. 「展覧会作成」画面を開く
2. 展覧会名・会期・会場・主催者メールアドレス等を入力
3. 必要に応じて「**🧪 練習モードで試す**」にチェック (詳細は §1.5 参照)
4. 「登録」 → 申請完了
5. 主催者メールアドレス宛に確認メールが届く → リンクをクリック → セットアップ画面へ
6. セットアップ画面で展覧会コード等を入力 → Submit
7. 完了画面で発行された展覧会コード `ex_code` を控える (例: `TE001`)
8. 完了通知メールが届く (URL リンクが入っている)

> ここで入力した「主催者メールアドレス」が、以降この展覧会のデータにアクセスできる人のアドレスになります。後で「データ書き出し」画面でログインする際もこのアドレスを使います。

## 1.5. 🧪 練習モード (おすすめ: 初めて使う方は必ず)

いきなり本番の展覧会で運用するのは不安、という方向けに練習モードがあります。

### 申請時に「練習モードで試す」にチェック

すると以下が変わります:

- 確認メール・完了メールに「[練習モード]」プレフィックス
- 各画面 (作品登録/設定・キャプション・データ書き出し) の上部に **「🧪 練習モード — 残り N 日で自動削除」** バッジが表示される
- 来場者・作家の画面にも「🧪 これは練習用展覧会です」バナーが表示される
- 「📨 案内メールを送信」を押すと送信前に「実在の作家へ転送禁止」確認ダイアログが出る
- 案内メールの件名に「[練習モード - 転送禁止]」が付き、本文先頭にも警告
- 作家が input.html で作品を新規登録しようとすると「これは練習用です。14 日後に削除されます」と確認ダイアログ

### 14 日後に自動削除

申請から 14 日経過すると自動的に削除されます。削除前日にも通知メールが届きます。

削除されるもの:
- 展覧会レコード (主催者の管理画面から見えなくなる)
- テスト作品データ・コメント・いいね
- 関連する Google Drive フォルダ・スプレッドシート

### 🎓 「本番運用に切替」(卒業) ボタン

練習で作った設定を気に入って、そのまま本番運用したい場合:

1. 「作品登録/設定」画面の「⚙️ 項目設定」タブを下までスクロール
2. 「🎓 本番運用に切替 (テストデータをクリア)」ボタンが表示されている
3. 押すと確認ダイアログ → OK
4. **テスト作品・コメント・いいね・画像ファイルがすべて削除される**
5. 設定 (項目・キャプションテンプレート等) はそのまま残る
6. `is_sandbox` フラグが解除され、自動削除予定もキャンセル
7. 真っ新な状態で本番運用を開始できる

### よくある質問

**Q. 練習中に作家へ案内メールを送ってしまったら?**
A. 件名と本文に「[練習モード - 転送禁止]」が付くので、転送するとは思いますが念のため: 案内メール自体は運営者宛のみに届くため、「転送ボタン」を押さない限り作家には届きません。万一転送した場合は、作家側の入力画面にもバナーが出るので大事故にはなりにくい設計です。

**Q. 練習モードで実際の QR 印刷を試したい**
A. 印刷自体は可能ですが、QR を会場に貼ると来場者が練習データに感想を書き込んでしまいます。練習モードでは QR 印刷の動作確認のみに留め、実際の貼り出しは本番運用に切替えてからにしてください。

**Q. 14 日経つ前にもう少し試したい**
A. 一度卒業ボタンで本番運用に切り替えれば自動削除はキャンセルされます (= ずっと残る)。気に入らなければ Firebase Console から手動削除も可能。

## 2. 項目設定をする

1. 「作品登録/設定」画面を展覧会コード `ex_code` 指定で開く
2. 「管理者ログイン」カードが出るので、運営者管理者メールアドレスを入力 → 「リンクを送信」
3. 届いたメールのリンクをクリック → 自動でログイン
4. 「⚙️ 項目設定」タブで使用する項目を選択 (チェック ON / 必須マーク / ドラッグで並び替え)
5. 「💾 保存する」をクリック

> 一度ログインすれば同じブラウザでは数日間ログイン状態が維持されます。毎回メールリンクを開く必要はありません。

### 過去展覧会から流用する場合
- 「📥 他の展覧会から項目設定を読み込む」をクリック
- 検索ボックスで絞り込み → 該当展覧会の「読み込む」
- 必要に応じて編集して「💾 保存する」

---

## 3. 作家へ案内メールを送る

1. 「⚙️ 項目設定」タブの下方「📨 案内メールを送信」のセクションへ
2. **「作家名」欄** に名前を入れるかどうかで URL の挙動が変わる:
   - **空欄** → 全作家共通の URL (作家が画面で作家名を入力して開始)
   - **入力あり** (例: `田中太郎`) → その作家専用 URL (作家名が自動入力される。別作家の作品には触れない)
3. 「📨 案内メールを送信」をクリック → 管理者宛にテンプレートメールが届く
4. それを編集して各作家へ転送

### 作家別 URL の使い分け (推奨)

応募が確定している作家のリストがある場合は、**作家ごとに 1 通ずつ** メールを生成するのがおすすめです:

- 作家別 URL なら作家名のスペル違い・取り違え事故が起きない
- 各作家にプライバシーがある (他の作家の登録状況が見えない)

応募の枠だけ先に決めて作家名が未確定の場合は共通 URL でも OK。

### 作品枠を追加
- 同じ画面の「作品枠を追加」で枠数を入れて「追加」
- 指定した数だけ空き作品枠 (作品 ID `artwork_id` 採番済み) が事前予約される
- 作家が新規登録すると、空き枠の 1 つに自動で割り当てられる

## 3.5. 🔒 作品をロックする (会期直前の入力締切)

会期が近づいて「もう変更しないでください」と作家に伝えたいとき、または会期中に内容を凍結したいときに使います。ロック中は作家側 (作品入力画面) では各項目が灰色になって編集できなくなります。

### 個別ロック (作家側からの自主提出)

作家が作品入力画面で **「✓ 提出する」** ボタンを押すとその作品 1 件だけがロックされます。「自分はもう編集しません」という意思表示。

### 主催者側で個別ロック / 解除

「作品登録/設定」画面 → 「📋 作品一覧」タブ → 該当作品を展開 → 編集パネル末尾の **🔒 / 🔓 ボタン**

### 一括ロック / 解除

「⚙️ 項目設定」タブの中段「**作品の一括ロック**」セクション:

- **🔒 全作品をロック**: 登録済の全作品を一気にロック (会期直前に推奨)
- **🔓 全作品をロック解除**: 全作品を解除 (会期後の修正受付・トラブル対応時)

> 会期当日は一括ロックしておくと、来場者・SNS 公開後に作家が誤って画像を差し替えてしまう事故を防げます。

## 4. 感想 (コメント) 受付期間の設定

1. 「⚙️ 項目設定」タブの下方「感想 (コメント) 受付期間」
2. 開始/終了の日時を入力 (空欄なら常時受付)
3. 「保存」
4. 来場者の感想画面では、期間外になると入力欄の代わりに「感想の受付期間は終了しました。」(または開始前なら開始日時のお知らせ) が表示される

---

## 5. キャプション/QR を印刷する

1. 「キャプション」画面を展覧会コード `ex_code` 指定で開く
2. 「管理者ログイン」カードが出ている場合はメールリンク認証 (§2 と同じ手順)。ログイン済みなら直接展覧会データが Load される
3. 「🖨️ Caption Print」タブで:
   - 「🎨 Templates」タブからテンプレートを選択 (📋 Standard / 🌐 Bilingual / 🎨 Print 等) → 「⚙️ Customize this template →」
   - 「⚙️ Custom」タブで項目の並び順・サイズ等を編集
   - 用紙サイズ・余白・QR 位置を設定
   - 「印刷」で PDF プレビュー → ブラウザで印刷
4. テンプレを保存しておきたい場合は「Template name...」に名前を入れて「💾 Save」
   - 同名で保存しようとすると上書き確認ダイアログが出る
5. 「🔲 QR Print」タブで QR コード一括印刷
6. 過去テンプレを使い回す場合は Saved templates から「Load」(展覧会またぎで使用可)

## 5.5. 🌐 Web 展覧会 (gallery) を公開する

会場に来られなかった人や会期後のアーカイブ閲覧用に、作品をオンラインで一覧できる **Web 展覧会** (gallery.html) を公開できます。

### 5.5.1. Web 展覧会設定画面を開く

- URL: `https://rohei-printer-system.web.app/web-exhibition.html?ex=○○○`
- 主催者メールアドレスでログイン

### 5.5.2. 公開設定 (visibility / interactions)

「公開設定」カードで 2 つの軸を設定:

**公開状態 (visibility)**:
| 値 | 動作 |
|---|---|
| 🔒 非公開 (closed) | 誰も見られない (デフォルト)。作業中・準備中はこれ |
| 🎫 QR 経由のみ (visitor_only) | 主催者が発行した会場 QR / リンクを持つ人だけ閲覧可 (会期中の招待制運用) |
| 🌍 公開 (public) | URL を知っていれば誰でも見られる (SNS 拡散・アーカイブ運用) |

**来場者の操作 (interactions)**:
| 値 | 動作 |
|---|---|
| 閲覧のみ (none) | いいね・コメント機能なし |
| いいねのみ (likes_only) | ハートボタンだけ |
| いいね + コメント (likes_and_comments) | 💬 でコメントも可 |

「💾 設定を保存」で反映。

### 5.5.3. ギャラリー外観

「🎨 ギャラリー外観」カードで gallery.html の見た目をカスタマイズ:

- **タイトル / サブタイトル**: SNS シェア時の OG プレビューにも使われる
- **ヒーロー画像**: ページ上部 + SNS プレビューの大画像 (公開展覧会では**設定推奨**)
- **背景画像 / 背景色**: 雰囲気作り (背景画像が優先)

「💾 外観を保存」で反映。

### 5.5.4. 会場 QR コードの発行 (visitor_only モード用)

visitor_only モードで使う有効期限付き URL を発行:

1. 「🎫 会場 QR コードの発行」カードへ
2. 有効期限 (日数) を入力 (デフォルト 60 日)
3. 「QR を発行」 → URL と QR 画像が表示される
4. ダウンロード or 印刷して会場に貼る or DM 等で配布

> public モードならこの操作は不要 (`gallery.html?ex=○○○` だけで誰でも見られる)。

### 5.5.5. SNS 拡散

public モードに切り替えた後の SNS 告知方法は別マニュアル **`manual_sns_promotion.md`** を参照してください。

## 6. 出力・レポート (ハブ画面)

展覧会のリアルタイム表示・データダウンロードへの入口を 1 つにまとめた画面です。「作品登録/設定」画面・「キャプション」画面のヘッダにある **「📊 出力・レポートへ」** から開けます (展覧会コードと認証は引き継がれます)。

ハブには 2 つの出力先があります:

### 📈 リアルタイムダッシュボード
- 参加者数 / いいね数 / コメント数 / 登録作品数の big number
- **いいねランキング 3 分割**:
  - 🏆 **総合**: 会場 QR + Web 展覧会 の合算
  - 🏛️ **会場**: 物理 QR からのいいねのみ
  - 🌐 **Web**: gallery.html (Web 展覧会) からのいいねのみ
- 最新コメントフィード
- 会場控え室モニタ等に常時表示する用途
- 「→ 開く (別タブ)」をクリックすると新しいタブで開く

### 📥 データダウンロード
- 「→ 開く」をクリックすると展覧会選択済みの状態で開く
- ダウンロードできる種類:
  - 🎨 **Artworks XLSX**: 作品データ。Excel で型崩れせず往復編集可
  - 💬 Comments CSV: コメント全件 (時刻順)
  - 👍 Likes CSV: いいね一覧 (session × 作品で dedup)
  - 📊 Summary CSV: 作品別 like/コメント数集計

> ハブ画面・ダウンロード画面ともに、主催者本人 (展覧会作成時に登録したメールアドレス) でログインしないと閲覧できません。

> §1 で展覧会を作る時に入力した「主催者メールアドレス」と、ここでログインするアドレスが**完全に一致**している必要があります。一致していないと「展覧会がありません」と表示されてダウンロードできません。

## 7. トラブルシューティング

- **ログインリンクのメールが届かない**: 迷惑メールフォルダを確認してください。Gmail なら「迷惑メールではない」を一度押すと、以降は受信トレイに届きます。
- **ログインリンクをクリックしたら「expired」「invalid」と出る**: リンクには有効期限 (約 1 時間) があります。もう一度メールリンクを送信してください。また、Outlook 等が先回りでリンクをプレビュー取得してしまうケースもあるので、メールアプリのリンクプレビューを切るか、別のメーラーで開くと回避できます。
- **「展覧会がありません」と表示される (データ書き出し画面)**: ログインしたメールアドレスと、展覧会作成時に登録した主催者メールアドレスが一致していない可能性。Firebase Console で当該展覧会の `email` フィールドを確認してください。
- **テンプレ Load しても項目が出ない**: 現在の展覧会の「項目設定」で当該項目が ON になっていない可能性。項目設定タブで確認。
- **画面更新が反映されない**: ブラウザキャッシュが古い可能性。`Ctrl+Shift+R` (Mac: `Cmd+Shift+R`) で強制再読み込み。
- **作品枠を増やしたい**: 「項目設定」タブ末尾「作品枠を追加」
- **過去展覧会の設定を全部使いたい**: 項目設定タブ「📥 他の展覧会から項目設定を読み込む」で復元 → 「キャプション」画面の Saved templates から該当テンプレを Load
- **作家から「ロックされていて編集できない」と連絡が来た**: 個別解除なら「作品一覧」タブから該当作品を展開して 🔓 ボタン、全件解除なら「項目設定」タブの「🔓 全作品をロック解除」
- **Web 展覧会の URL を SNS に貼ってもプレビュー画像が出ない**: web-exhibition.html → ギャラリー外観 で **ヒーロー画像** を設定。設定後は Facebook Sharing Debugger 等でキャッシュをクリア
- **gallery が「閲覧には QR コード経由のアクセスが必要です」と表示される**: visibility が `visitor_only` のまま。SNS 拡散したいなら `public` に切替
