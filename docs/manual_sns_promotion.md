# SNS 告知マニュアル (Instagram / Facebook)

バーチャル展覧会 (`gallery.html?ex=<コード>`) の URL を SNS で告知する方法をまとめます。Instagram と Facebook では URL の扱いがかなり違うので、それぞれ整理しています。

---

## 0. 前提

- web-exhibition.html で **公開状態 = public** にしておく (誰でも URL からアクセス可能な状態)
- ヒーロー画像 / タイトル / サブタイトルを設定しておくと SNS 上でリッチなプレビューカードになる (web-exhibition.html → ギャラリー外観)
- gallery URL は `https://rohei-printer-system.web.app/gallery.html?ex=<展覧会コード>` の形

---

## 1. Instagram

Instagram は**フィード投稿のキャプションに貼った URL がタップできない**ため、リンク誘導には工夫が必要。

### 1.1 ストーリーズ + リンクスタンプ (もっとも効果的)

1. Instagram アプリを開く → 自分のアイコンから **ストーリーズを作成**
2. 画像 / 動画を追加 (展覧会の雰囲気が伝わるもの。ヒーロー画像のスクショでも可)
3. 上部のスタンプアイコン → 「**リンク**」を選択
4. URL 欄に gallery URL を貼り付け
5. 「カスタマイズ (任意)」でボタンの文言を変更 (例:「展覧会を見る」)
6. 「完了」 → スタンプを画面に配置 → 「ストーリーズに投稿」
7. 視聴者がタップするとブラウザで gallery が開く

24 時間で消えるが、フォロワーへの即時告知に最適。

### 1.2 プロフィール bio にリンク (常時表示)

1. Instagram → 自分のプロフィール → 「**プロフィールを編集**」
2. **ウェブサイト**欄に gallery URL を貼り付け → 保存
3. プロフィール画面に常時リンクが出る
4. フィード投稿のキャプションには「**詳細はプロフィールのリンクから**」と書いて誘導

複数の展覧会 / 複数のリンクをまとめたい場合は **Linktree** (https://linktr.ee) や **Bento** (https://bento.me) などのリンク集サービスを bio に置いて、その中に gallery URL を入れる運用が一般的。

### 1.3 DM (個別 / グループ送信)

1. 送りたい相手の DM 画面
2. URL をペースト → 送信
3. 自動でプレビューカードが表示される

少人数の知人に直接送るときに手軽。

### 1.4 フィード投稿 + bio 経由

1. 展覧会の写真 (会場 / 作品 / フライヤー) をフィードに投稿
2. キャプション例:

   > バーチャル展覧会「Noboru' Story」公開中。
   > 会場に来られなかった方もぜひオンラインでご覧ください。
   >
   > 👉 詳細はプロフィールのリンクから

3. プロフィール bio のリンクから流入

### 1.5 シェアボタン経由 (gallery.html から)

スマホで gallery を開く → 右下「⤴」ボタンタップ → 共有シートで **Instagram** を選択 → ストーリーズ or DM の編集画面に URL 入りで遷移。

### 推奨運用

- **告知時**: 1.2 (bio リンク設定) + 1.1 (ストーリーズ) + 1.4 (フィード投稿)
- **会期中**: 反応のよかった作品を引用してストーリーズに繰り返し投稿。毎回リンクスタンプを付ける

## 2. X (旧 Twitter)

X は **URL をそのまま投稿に貼れる**ため告知が簡単。日本語 140 文字制限がポイントで、URL は t.co で短縮されて約 23 文字相当としてカウントされる。

### 2.1 通常の投稿 (ポスト)

PC でもスマホでも手順は同じ:

1. 「ポストを作成」欄を開く
2. 文章を入力 (例:「バーチャル展覧会を公開しました。会場に来られなかった方もぜひ。 #個展 #バーチャル展覧会」)
3. 改行して URL を貼り付け
4. **数秒待つと URL が OG プレビューカードに自動展開** (ヒーロー画像 + タイトル + サブタイトル)
5. プレビューが出たら本文の URL テキストは削除可 (カードだけ残る)
6. 「ポスト」

ハッシュタグ (例: `#個展` `#バーチャル展覧会` `#美術` `#現代アート`) を 1〜3 個入れると検索流入が増える。

### 2.2 ピン留め投稿

会期中はプロフィールページ最上部に告知を固定すると効果的:

1. 2.1 の手順で告知ポストを投稿
2. 投稿右上の「⋯」メニュー → 「**プロフィールに固定**」
3. プロフィール最上部に常時表示される
4. 会期後はピン留めを外す or 別ポストに差し替え

### 2.3 プロフィール bio + ウェブサイト欄

1. プロフィール → 「**プロフィールを編集**」
2. **自己紹介**または**ウェブサイト**欄に gallery URL を入力
3. ウェブサイト欄は専用のリンクとして表示、自己紹介欄の URL も自動でリンク化される

### 2.4 スレッド (一連の投稿で詳細を伝える)

長めの告知や会期中の進捗には:

1. まず告知ポスト + URL を投稿
2. 自分の投稿に「返信」して連投 (テーマ / 注目作品 / 来場者の声 など)
3. 画像と文章で展覧会のストーリーを伝えられる

### 2.5 DM

1. DM 画面で URL をペースト → 送信
2. 自動でプレビューカード化される

少人数への個別連絡向け。

### 2.6 シェアボタン経由 (gallery.html から)

スマホで gallery.html を開く → 右下「⤴」ボタン → 共有シートで **X** を選択 → ポスト編集画面に URL + タイトルが入った状態で遷移 → ハッシュタグを加えて投稿。

### 2.7 予約投稿 (X Premium / 外部ツール)

- **X Premium (有料)**: ポスト作成欄のカレンダーアイコン → 投稿日時を指定 → 「予約投稿する」
- **無料アカウント**: 外部ツール (Buffer / Hootsuite / TweetDeck の代替) を併用

### 推奨運用

- **会期前**:
  1. 2.3 プロフィール bio + ウェブサイト欄に URL
  2. 2.1 + 2.2 告知ポストを作成し、ピン留め
  3. ハッシュタグを工夫 (`#個展` `#バーチャル展覧会` `#現代アート` など)
- **会期中**:
  - 2.4 スレッドで進捗発信 (反応のよかった作品 / 運営の裏側 / 来場者の声)
  - 引用リポストで反応を可視化
- **会期後**:
  - ピン留めを「ご来場ありがとうございました」ポストに差し替え (gallery URL は引き続き含める)

### 注意点

- **文字数**: 日本語 140 字制限。URL 短縮分を考慮して本文を組む
- **OG キャッシュ**: 30 分〜数時間で更新される (Facebook ほど執拗ではない)
- **画像なしカード**: `og:image` が無いと小さなテキストカードになる。ヒーロー画像必須
- **クリック解析**: X Analytics でピン留め投稿のインプレッション / リンクのクリック率を確認できる

## 3. Facebook

Facebook は **URL をそのまま貼って投稿できる**ため、Instagram より告知が簡単。

### 3.1 フィード投稿 (個人 or ページ)

PC でもスマホでも手順は同じ:

1. 「投稿を作成」欄を開く
2. 文章を入力 (例:「バーチャル展覧会を公開しました。会場に来られなかった方もぜひ。」)
3. 改行して URL を貼り付け
4. **数秒待つと URL が OG プレビューカードに自動展開** (ヒーロー画像 + タイトル + サブタイトル)
5. プレビューカードが出たあと、本文の URL テキストを削除すると見栄えが整う (カードだけ残る)
6. 「投稿」

これが基本形。

### 3.2 Facebook ページ (展覧会 / 主催者として)

個人ではなく Facebook ページを持っている場合:

1. ページ管理画面で「投稿を作成」
2. 同じ流れで URL を貼る
3. 「**今すぐ投稿**」または「**スケジュール設定**」(会期初日朝に自動投稿予約も可)
4. **Meta Business Suite** (https://business.facebook.com) を使えば Instagram への同時投稿もここから一発でできる

メリット: 過去投稿がアーカイブとして残る、リーチ / シェア数が見える、広告化できる。

### 3.3 ストーリーズ + リンクスタンプ

Instagram と同じ手順:

1. Facebook アプリ → ストーリーズを作成
2. 画像を追加
3. ステッカー → リンク → URL 貼り付け
4. 投稿

24 時間で消えるが即時告知に向く。

### 3.4 Messenger (個別 / グループ送信)

1. Messenger のトーク画面
2. URL をペースト
3. 自動でプレビューカード化されて送信

LINE と同じ感覚。

### 3.5 グループ投稿

地域コミュニティ / アート関連 / 主催者ネットワーク等の Facebook グループに所属している場合:

1. グループのページに入る
2. 「投稿を作成」で URL を貼る
3. 投稿 (管理者の承認制グループの場合は承認待ち)

ターゲットが絞れている分クリック率が高い。

### 3.6 イベント機能と組み合わせ

物理会場のイベントを Facebook イベントで立てている場合:

1. イベント詳細欄に gallery URL を貼る
2. イベントの「ディスカッション」タブでも投稿可
3. RSVP した参加者にリンクが届く

### 3.7 シェアボタン経由 (gallery.html から)

スマホで gallery を開く → 「⤴」ボタン → 共有シートで Facebook → 投稿編集画面に URL 入りで遷移 → 投稿。

### 推奨運用

- **会期前**:
  1. Facebook ページで告知投稿 (なければ個人タイムライン)
  2. イベント作成 + 詳細欄に gallery URL
  3. 関連グループにも投稿
- **会期中**: ストーリーズで新作 / 来場者の声を 24h アップしつつ、反応のよかった内容をフィードに再投稿

## 4. プレビュー (OG タグ) の確認と更新

### 確認ツール

| ツール | URL | 特徴 |
|------|-----|------|
| opengraph.xyz | https://www.opengraph.xyz/ | ログイン不要・各 SNS のプレビューを横並びで一覧 |
| Facebook Sharing Debugger | https://developers.facebook.com/tools/debug/ | Facebook 公式 / 詳細な OG 取得結果 / キャッシュ更新可 |
| metatags.io | https://metatags.io/ | 汎用 / Twitter / Facebook / Pinterest / Slack / Google 検索プレビュー |
| LINE Keep メモ | LINE 内の自分専用トーク | 実機での見た目を確認できる |

### キャッシュ注意点

- Facebook は OG タグを強くキャッシュする。タイトル / 画像を変更しても**古いまま出ることが多い**
- 解決方法: Facebook Sharing Debugger で URL を入力 → 「**もう一度スクレイピング**」ボタンを押す
- LINE も似たキャッシュがある。URL 末尾に `&_=2` のようなダミーパラメータを付けて別 URL 扱いにすると再取得される

### OG タグの中身 (現状)

`gallery.html?ex=<コード>` を開いたとき、`exhibitions/{ex}` から下記が反映される:

| OG フィールド | 取得元 (優先順) | 備考 |
|---|---|---|
| `og:title` / `<title>` | `gallery_title` → `ex_name` → コード | web-exhibition.html → ギャラリー外観 → タイトル |
| `og:description` | `gallery_subtitle` | web-exhibition.html → サブタイトル |
| `og:image` | `gallery_hero_url` | web-exhibition.html → ヒーロー画像 (未設定なら出さない) |
| `og:url` | `https://rohei-printer-system.web.app/gallery.html?ex=<コード>` | sig / exp は除外 |
| `og:site_name` | "My Art Fair" | 固定 |

ヒーロー画像が未設定だと OG プレビューに画像が出ないので、SNS 拡散したい場合は**必ず画像を設定**する。

## 5. 推奨フロー (会期 1 回ぶん)

1. **会期 1〜2 週間前**:
   - web-exhibition.html で公開状態 = public、タイトル / サブタイトル / ヒーロー画像 / 背景を整える
   - opengraph.xyz でプレビュー確認
   - X / Facebook ページとイベントを準備、Instagram と X の bio リンクを更新
2. **会期初日朝**:
   - X 告知ポスト + プロフィール固定 (ピン留め)
   - Facebook フィード投稿 (個人 / ページ / 関連グループ)
   - Instagram ストーリーズにリンクスタンプ
   - Instagram フィード投稿 (キャプションで bio リンク誘導)
3. **会期中**:
   - X スレッドで進捗発信 (反応のよかった作品 / 運営の裏側 / 来場者の声)
   - 反応のよかった作品を Instagram ストーリーズで再投稿
   - 来場者の感想 (許可をとったうえで) を引用シェア
   - Facebook イベントの「ディスカッション」で進捗共有
4. **会期終了後**:
   - X のピン留めを「ご来場ありがとうございました」投稿に差し替え (gallery URL は引き続き含める)
   - public のまま残してアーカイブ運用 (恒久 URL として SNS から参照可能)
   - もしくは web-exhibition.html で closed に切り替えて停止
