# 変更履歴 (Changelog)

このファイルは「展覧会システム」の主催者向け視点の更新履歴です。

バージョン体系: semver-lite
- `v0.x.y` — プレリリース、関係者テスト期
- `v1.0.0` — 公開リリース
- `v1.X.0` — 機能追加
- `v1.X.Y` — バグ修正・軽微な改善
- `v2.0.0+` — 破壊的変更

## v0.18.0 — 2026-06-26

### 改善（作品リスト）

- **列幅をデータから自動調整**。「📐 データから列幅を自動調整」を押すと、登録済みの作品の中身に合わせて各列にちょうどよい幅を割り当てます。各列の「幅」欄で手直しもできます。
- **名前を付けたリストを複数**作れるように。1 つの展覧会で「価格表」「作品目録」「受付台帳」のように、載せる項目やレイアウトの違う表を別々に保存して切り替えられます。
- **段組でも、長いリストが自動でページに分かれて続くように**。各段の先頭に見出しが付き、途中で切れません（2〜3 段のとき）。

### 改善（使いやすさ・特に作家向け）

- **作家の招待を「招待リンク＋案内文をコピー」方式に変更**。これまでは案内メールがいったん主催者ご自身に届いて転送する必要がありましたが、「📝 案内を作成」で招待リンクと案内文が表示され、LINE・メール・DM など普段の連絡手段でそのまま作家へ送れます（メールでご自身に送る方法も残しています）。
- **作家が「✓ 提出する」を押すと、完了メッセージが画面上部に表示**されるように。提出できたか分かりにくい問題を解消しました。
- **作家の入力画面の冒頭に「はじめての方へ」の手順**（①名前を入力 ②作品を追加 ③提出）を追加。初めての作家が迷いにくくなりました。
- **エラー時の表示を分かりやすい日本語に統一**。英語の専門的なメッセージが出ていた箇所を整理し、作家向けには「解決しないときは主催者にご連絡ください」と案内を添えました。

## v0.17.1 — 2026-06-23

### 改善・修正（キャプション）

- **作家コメント等の改行が、改行されたまま印字されるように**。複数行で入力した内容が 1 行に繋がって長くなり、QR コードと重なって文字が消える問題を解消しました。
- **QR コードを下に置いたときも、文字が QR をよけて回り込めるように**。QR 位置に「左下に折り返し」「右下に折り返し」を追加しました（これまでは上の左右だけでした）。
- **ページ設定タブから直接保存できるボタンを追加**。用紙・カードシート等のページ設定を、③詳細編集タブに移動しなくても、その場で「この展覧会のテンプレとして保存」できるようにしました（保存しないと次回戻ってしまう導線を改善）。

## v0.17.0 — 2026-06-23

### 新機能

#### 作品リスト出力（紙 / PDF の一覧表）

- 出展作品の一覧表を作って印刷できるようになりました（「出力・レポート」画面の「📋 作品リスト」から）。価格表・作品目録・受付台帳などに使えます。
- **リストのタイトル**、**表示する項目と列の順番**、**見出しの配置（左寄せ / 中央）** を選べます。
- **作品の並び順**をドラッグで自由に指定できます（このリスト印刷用。作品 ID は変わりません）。
- **用紙（A4 / Letter）・向き（縦 / 横）・段組（縦は最大 2 段 / 横は最大 3 段）** に対応。1 ページにより多くの作品を載せられます。
- 設定は展覧会ごとに保存されます。

#### キャプションのカードシート印刷（エーワン等の型抜きカード）

- 型抜きカードシートに合わせて、**カードの幅・高さ・上下左右の余白・カード間隔（mm）** を指定して印刷できるモードを追加しました（ページ設定タブ）。
- **使うときだけ ON** にする方式で、OFF のときは従来とまったく同じ印刷です（これまでの使い方は何も変わりません）。
- カードシート印刷のときは、キャプションの枠線を自動で消します（カードの縁に合わせるため）。

### 改善・修正

- **キャプション：未入力の項目が印刷に出ないように修正**。公式テンプレ使用時、作品で入力していない項目に表示される参考用のグレー文字が、印刷にもそのまま出てしまう問題を直しました（画面と印刷が一致するように）。
- **キャプション：項目を選ぶ途中の警告ポップアップを解消**。「載せる項目」を 1 つずつ選んでいる間に確認ポップアップが繰り返し出ていたのをやめ、プレビュー欄の静かな案内に変えました。
- **作品登録：必須項目のエラーを分かりやすく**。作家名が未入力のとき、どの項目が足りないかを表示し、作家欄を自動で開くようにしました。
- **作品登録：登録した作品がすぐ一覧に表示されるように**（再読み込み不要）。
- **作品登録：価格欄の桁ずれを修正**。日本語入力の確定時に価格の桁が増えてしまうことがある不具合を直しました。
- **練習モードの卒業時にデータの扱いを選べるように**。練習で入れた作品・コメント・いいねを「残して本番化」「消して本番化」から選べるようになりました。
- **内部改善**：作品枠の発番・作成のしくみを整理しました（操作感は変わりません）。

## v0.16.0 — 2026-06-15

### キャプション作成画面の刷新（初心者向け UX）

キャプション作成まわりを、初心者でも迷わず使えるように一通り整理しました。

#### 言葉と画面の分かりやすさ
- メインタブ・見出しを日本語に統一（「Caption Print」→「キャプション印刷」など）。
- 設定タブを使う順に番号化：**① テンプレート → ② 載せる項目 → ③ 詳細編集 → ④ ページ設定**。最初に「おすすめの手順」を表示し、③④は「必要なときだけ」と明記。
- 「フォーム設定」を「**キャプションに載せる項目**」に改名し、説明を実態に修正。**作品登録（＝作家から集める項目）／キャプション（＝そのうち載せる項目）**の役割を画面の言葉で区別。

#### テンプレートの扱いを安全・確実に
- テンプレートを**展覧会ごと**に保存（別名で複数保存でき、別の展覧会と名前が同じでも混ざりません）。一覧を「**この展覧会のテンプレ**」「**過去の展覧会から**（元の展覧会名つき）」に整理。
- 過去の展覧会のテンプレを**読み込むと、その展覧会用に項目・設定が一式整う**ように。読み込み後の保存名は展覧会名が入り、元のテンプレを誤って上書きしません。
- 公式テンプレを採用しても**他の保存テンプレは壊れません**（採用は「試す」操作）。公式を使っている間は「💾 この展覧会のテンプレとして保存」の案内を表示。
- 設定の複製（他の展覧会から）でも、テンプレがこの展覧会用に正しくコピーされるよう修正。保存テンプレの削除が効かない不具合も修正。

#### 「載せる項目」まわり
- 詳細編集には「載せる項目」で選んだ項目だけが並ぶように（作品登録を増やしても画面が縮みません）。②で項目を増減すると③にも反映。
- 「② 載せる項目」は**いま集めている項目だけ**を、作品登録と同じ並びで表示。ここに無い項目は「作品登録・設定」で追加する案内付き。
- 不具合修正：キャプション保存テンプレートが、保存しても再読み込みで戻ってしまう問題を修正。

#### スマホでの利用
- 細かいレイアウト編集はPC向きと割り切り、スマホで「③ 詳細編集」を開いたときに「細かい調整はPC画面が楽です」と案内（操作自体はスマホでも可能）。

## v0.15.0 — 2026-06-14

### Web 展覧会の公開期間（会期）設定

- **公開開始・公開終了の日時を設定できるように**：Web 展覧会の設定に「公開開始（任意）」「公開終了（任意）」を追加しました。公開状態が「公開」または「QR を持つ来場者のみ」のとき、この期間内だけ閲覧できます。会期に合わせて自動で公開・終了できるので、当日まで非公開→会期中だけ公開→終了後は自動クローズ、という運用が手動操作なしで行えます。
  - 公開前にアクセスすると「🕒 まだ公開されていません（◯◯から公開予定です）」、終了後は「🔚 公開は終了しました」と来場者に表示されます。
  - 両方とも空欄なら従来どおり制限なし（常時公開）。「非公開」は期間に関係なく常に非公開のままです。

### 不具合修正

- **キャプションの保存テンプレートが正しく復元されない不具合を修正**：詳細編集で項目（価格・制作年など）のチェックを外して上書き保存しても、再読み込み時にチェックが戻ってしまう問題を修正しました。外した項目は外したまま復元されます。

## v0.13.0 — 2026-06-13

### セキュリティ強化 (なりすまし・無認証経路・XSS の封じ込め)

Fable によるコード全体の精読レビューで見つかった脆弱性をすべて修正しました。利用者から見える機能変更はほぼありませんが、不正操作の経路を大きく塞いでいます。主催者・運営者・来場者の通常操作には影響しません。

#### GAS 無認証経路の鍵かけ
申し込み・初期設定・メール送信・データ改変などを行う裏側プログラム (GAS) が、展覧会コードさえ分かれば誰でも運営者権限で叩ける状態でした。Cloud Function (`callGasAuthed`) を「受付」として挟み、Firebase ログイン + 主催者 / 運営者の権限確認を通してから共有秘密 (ADMIN_SECRET) 付きで GAS を呼ぶ形に統一。GAS 側も対象操作を秘密必須にして直接叩きを封じました。ログイン不要な経路 (問い合わせ・申し込み) は回数制限で連投を抑制。

#### Web 展覧会ページの XSS 修正
作家が入力した作品名・画像 URL 等に細工をすると、Web 展覧会ページ (gallery) に不正なスクリプトを仕込める状態でした。文字のエスケープ処理 (属性コンテキストで `"` `'` が残る不具合) を修正して封じました。あわせて他画面の文字表示処理も安全な実装に統一。

#### いいね・コメントの取消なりすまし対策
取消・削除の本人確認を、公開される識別子 (sessionId) ではなく、本人のブラウザだけが持つ秘密の鍵 (ownerKey) で行う方式に変更。鍵の指紋 (sha256) だけをデータに保存し、取消は Cloud Function (`deleteLike`) が照合してから実行します。他人のいいね・コメントを消せる穴を塞ぎました。

#### 軽微なバグ修正
- いいねデータの各項目にサイズ上限 + 想定外フィールド禁止を追加 (スパム抑制)
- 展覧会コードが数字だけのとき先頭ゼロが消えるバグを修正 (自動生成も英字付きに)
- 本番運用への切替が途中で失敗したときの再試行案内を追加

## v0.12.0 — 2026-05-27

### セキュリティ強化 + AI-Native アーキテクチャ整備

利用者から見える機能変更はありませんが、データ保護とサーバ側ルールを大きく整備しました。主催者・運営者の通常操作には影響しません。

#### D-1: 作品画像の上書きを Cloud Function 経由に
従来、`/artworks/` や `/gallery/` の Storage にファイル名さえ分かれば誰でも画像を上書きできる構造でした。Cloud Function (`uploadArtworkImage` / `uploadGalleryImage`) 経由に倒し、Storage Rules を読取り専用に。アップロード時に主催者 / 運営者 / 招待 URL の token が検証されるようになりました。

#### D-2: AI 分類機能 (categorizeArtwork) の権限強化
来場者の認証だけで AI 分類 API を叩ける状態だったのを、主催者 / 運営者のみに限定。Claude API 課金の抜け穴を塞ぎました。

#### D-3: サインインリンク送信に rate limit
任意のメールアドレスに対して大量の招待メールを送れる構造だったのを、5 分以内に同一アドレスへ 3 回までに制限。`email_throttle/{hash}` collection で server-side で管理。

#### D-6: 展覧会卒業 (graduateExhibition) の audit log 記録
展覧会全データを削除する破壊的操作の痕跡を `audit` collection に必ず残すように。`admin/audit.html` から閲覧可能。

#### D-7: inquiries の `from='admin'` 命名意図を明文化
firestore.rules と inquiry.html のコメントを補強。`from='admin'` が「問い合わせスレッドの作成者本人」を指す legacy 命名であることを明示。

#### β-3: 作品データの read 経路を整理 (AI-Native 不変条件の徹底)

`artworks` collection の公開 read を撤去し、3 経路だけに絞り込みました:
- **運営者**: Firebase Auth で常時 read
- **主催者**: 作品 doc に保存された `organizerEmail` と一致する Firebase Auth (denormalize)
- **来場者 (Web 展覧会)**: visitor custom token + 展覧会公開状態 (`_published=true`) + 登録済 (`status='1'`)

それ以外 (QR スキャンで来た来場者・招待 URL から来た作家) は Cloud Function (`getArtwork` / `listArtworksByArtist` / `findEmptyArtworkSlot`) 経由で取得するように変更。

これにより:
- 非公開状態の展覧会の作品データが exhibitionId + artworkId 既知でも漏れない
- LLM / 自動化ツールが Firebase SDK を直接叩いても不変条件を破れない
- 将来の公募展 (審査期間中は秘匿) / アートフェア (フェア前 VIP プレビュー) など、機密性要求の高い機能の土台

web-exhibition.html の「Web 公開設定」保存も Cloud Function (`syncArtworkPublishedFlags`) 経由になり、`gallery_visibility` 変更時に全作品の `_published` flag が一括同期されます。

### マイグレーション

新規展覧会 / 新規作品は自動的に新フィールドが入りますが、既存の作品 doc には `node functions/scripts/backfill-artwork-published.js --all` での一度きりのバックフィルが必要です (`--dry-run` で事前確認可)。

### 撤去された機能 (一時)

- index.html の「同じ展覧会の作家名 autocomplete」(SNS 自動入力に使われていた dropdown)。代替の Cloud Function を Phase 2 で追加すれば復活可能。

## v0.11.0 — 2026-05-26

### 新機能: QR のみキャプション (NY ギャラリースタイル)

紙キャプションの代わりに **小さな QR シール** を作品横に貼り、来場者が QR をスキャンすると **スマホでリッチな作品情報** + いいね/コメントが見える運用を追加。NY 系ギャラリーの「作品優先・説明排除」スタイルや、印刷コストを最小化したい主催者向け。

**新しい仕組み**:

- **公式テンプレ「📍 QR のみ (NY スタイル)」** を caption.html のテンプレ一覧に追加
- **paperMode 設定** (Page タブ、全テンプレ共通):
  - `full` (デフォルト): 紙にキャプション + QR (= 従来の挙動、変化なし)
  - `qr_only`: 紙には QR シールだけ、スマホで詳細表示
- どの公式テンプレも paperMode を切替えて「キャプション → QR シール運用」に転用可能 (例: 個展推奨で paperMode='qr_only' を選ぶ)
- **QR シールラベル** (Page タブの 5 択 select、qr_only 時のみ表示):
  - なし (QR のみ、QR をカード中央配置で間延び解消)
  - 作家名 / 作品 ID / 作家 + 作品 ID (default) / タイトル + 作家
- **Custom タブの items[] の意味が paperMode で切替わる**:
  - full モード時: 紙キャプションのデザイン (従来)
  - qr_only モード時: **スマホ画面 (= 来場者が見るキャプション)** のデザイン

**スマホ画面 (index.html caption mode)**:

- URL `?caption=1` (QR-only モードの QR シールに自動付与) で起動
- 画像 + items[] driven の作品情報 + ピンクの「いいね & 感想を送る」カード
- 紙キャプと同じ design language (font-size pt / bold / italic / divider / spacer / group / between 等) でスマホに表示

**preview の UX**:

- preview エリアにタブ追加: 「📄 紙印刷」/「📱 スマホ caption」
- paperMode='full' のときはスマホタブを **非表示** (= 普通の紙キャプ運用者は混乱しない)
- paperMode='qr_only' のときだけ両タブが見えて、デフォルトはスマホ caption

## v0.10.0 — 2026-05-25

### 新機能: 詳細分析画面 (`analytics.html`)

リアルタイムダッシュボード (`dashboard.html`) はライブ表示専用 (会場大画面用) として据え置き、深い分析は新規 `analytics.html` に分離。ライブで動く数値と、腰を据えて読む数値を画面ごとに役割分担。

**新規指標**:

- **visitor の質** (V-1〜V-3):
  - コメント率 / 平均コメント長 (中央値)

- **作家別** (G-1/G-3/G-4): グループ展のときだけ自動表示
