# 主催者向けマニュアル

展覧会を主催する方向けの手順書です。展覧会の作成からキャプション印刷、Web 展覧会の公開、データ書き出しまでを順番に説明します。

---

## 0. 用語と画面の流れ

- **展覧会コード** (例: `TE001`): 各展覧会につく識別番号です。展覧会作成時に発行されます。
- **主催者ログイン**: 各画面を開く際に必要です。メールアドレスを入力 → 届いたリンクをクリック、で完了。一度ログインすると同じブラウザでは数日間維持されます。

> 作家が使う「作品入力画面」と来場者が使う「作品ページ (QR から開く)」「Web 展覧会」はログイン不要です。

主な画面と用途:

| 画面 | 用途 |
| --- | --- |
| 展覧会作成 | 新規展覧会を申請する出発点 |
| 作品登録/設定 | 入力項目の設定・案内メール送信・作品枠追加・感想受付期間・作品ロック |
| 作品入力 (作家用) | 作家が自分の作品データを入力する画面 |
| キャプション | キャプション印刷・QR コード一括印刷・テンプレート管理 |
| ダッシュボード | いいね/コメントのリアルタイム表示 |
| データ書き出し | 作品データ・コメント・いいねを Excel / CSV で取得 |
| Web 展覧会の設定 | オンラインギャラリーの公開設定・外観・会場 QR 発行 |

よく使う画面はブラウザのブックマークに登録しておくのがおすすめです。

## 1. 新しい展覧会を作る

1. 「展覧会作成」画面を開く
2. 展覧会名・会期・会場・主催者メールアドレス等を入力
3. 必要に応じて「**🧪 練習モードで試す**」にチェック (詳細は §1.5 参照)
4. 「登録」 → 申請完了
5. 入力したメールアドレス宛に確認メールが届く → リンクをクリック → セットアップ画面へ
6. セットアップ画面で展覧会コード等を入力 → Submit
7. 完了画面で発行された **展覧会コード** を控えてください (例: `TE001`)
8. 完了通知メールが届きます (各種 URL 入り)

> ここで入力した「主催者メールアドレス」が、以降この展覧会を編集できる人のアドレスになります。「データ書き出し」画面でログインする際もこのアドレスを使います。

## 1.5. 🧪 練習モード (おすすめ: 初めて使う方は必ず)

いきなり本番運用するのは不安、という方向けの練習モードです。

### 申請時に「練習モードで試す」にチェック

すると以下が変わります:

- 確認メール・完了メールに「[練習モード]」が付く
- 各画面 (作品登録/設定・キャプション・データ書き出し) の上部に **「🧪 練習モード — 残り N 日で自動削除」** バッジが表示される
- 来場者・作家の画面にも「🧪 これは練習用展覧会です」バナーが表示される
- 「📨 案内メールを送信」を押す前に「実在の作家へ転送禁止」確認ダイアログが出る
- 案内メールの件名に「[練習モード - 転送禁止]」が付き、本文先頭にも警告
- 作家が新規登録しようとすると「これは練習用です。14 日後に削除されます」と確認ダイアログが出る

### 14 日後に自動削除

申請から 14 日経過すると自動的に削除されます。削除前日にも通知メールが届きます。

削除されるもの:
- 設定・テスト作品データ・コメント・いいね・画像ファイル、すべて消去
- 主催者の管理画面からも見えなくなります

### 🎓 「本番運用に切替」(卒業) ボタン

練習で作った設定を気に入って、そのまま本番運用したい場合:

1. 「作品登録/設定」画面の「⚙️ 項目設定」タブを下までスクロール
2. 「🎓 本番運用に切替 (テストデータをクリア)」ボタンが表示されています
3. 押すと確認ダイアログ → OK
4. テスト作品・コメント・いいね・画像ファイルがすべて削除されます
5. **設定 (項目設定・キャプションテンプレート等) はそのまま残ります**
6. 練習モード扱いが解除され、自動削除予定もキャンセルされます
7. 真っ新な状態で本番運用を開始できます

### よくある質問

**Q. 練習中に作家へ案内メールを送ってしまったら?**
A. 件名と本文に「[練習モード - 転送禁止]」が付くので、まず転送はしないと思いますが念のため: 案内メール自体は主催者宛のみに届くため、「転送ボタン」を押さない限り作家には届きません。万一転送した場合も、作家側の入力画面にもバナーが出るので問題は起こらない設計です。

**Q. 練習モードで実際の QR 印刷を試したい**
A. 印刷自体は可能ですが、QR を会場に貼ると来場者が練習データに感想を書き込んでしまいます。練習モードでは QR 印刷の動作確認のみに留め、実際の貼り出しは本番運用に切替えてからにしてください。

**Q. 14 日では試す時間が足りない**
A. 一度卒業ボタンで本番運用に切り替えれば自動削除はキャンセルされます。練習用バナーや警告ダイアログも消えるため、その状態で実運用に移行する形になります。もう一度練習モードで試したい場合は、新しい練習モード展覧会を改めて作成してください。

## 2. 入力項目を設定する

1. 「作品登録/設定」画面を展覧会コードを指定して開く
2. 「主催者ログイン」カードが出るので、登録時のメールアドレスを入力 → 「リンクを送信」
3. 届いたメールのリンクをクリック → 自動でログインされ、設定画面が開きます
4. 「⚙️ 項目設定」タブで使用する項目を選択 (チェック ON / 必須マーク / ドラッグで並び替え)
5. 「💾 保存する」をクリック

> 一度ログインすれば同じブラウザでは数日間維持されます。毎回メールリンクを開く必要はありません。

### 主催者ご自身が過去に実施した展覧会から設定を複製する

- 「📥 他の展覧会から設定を複製」をクリック
- パネルが開きます。複製したい内容のチェックボックスを選択:
  - **項目設定**: 作品入力フォームに出すフィールドの一覧
  - **キャプション設定**: キャプション印刷時の項目選択 + 最後に使ったテンプレート (キャプション画面で再選択不要)
  - **🖼️ Web 公開設定**: Web 展覧会の公開状態 / いいね・コメント許可 / ヒーロー画像 / 背景画像
- 検索ボックスで展覧会名を絞り込み → 該当展覧会の「複製」
- 必要に応じて編集して「💾 保存する」

---

## 3. 作家へ案内メールを送る

1. 「⚙️ 項目設定」タブの下方「📨 案内メールを送信」のセクションへ
2. 「作家名」欄に名前を入れるかどうかで挙動が変わります:
   - **空欄** → 全作家共通の URL になる (作家は画面で自分の名前を入力して開始)
   - **入力あり** (例: `田中太郎`) → その作家専用の URL が発行される (作家名が自動入力され、別作家の作品には触れない)
3. 「📨 案内メールを送信」をクリック → ご自身の主催者メールアドレスにテンプレートメールが届きます
4. それを編集して各作家へ転送してください

### 作家別 URL の使い分け (推奨)

応募が確定している作家のリストがある場合は、作家ごとに 1 通ずつ メールを生成するのがおすすめです:

- 作家別 URL なら作家名のスペル違い・取り違え事故が起きない
- 各作家のプライバシーが守れる (他の作家の登録状況が見えない)

応募の枠だけ先に決めて作家名が未確定の場合は共通 URL でも OK です。

### 作品枠を追加する

- 同じ画面の「作品枠を追加」で枠数を入れて「追加」
- 指定した数だけ空き作品枠が事前予約されます
- 作家が新規登録すると、空き枠の 1 つに自動で割り当てられます

## 3.5. 🔒 作品をロックする (会期直前の入力締切)

会期が近づいて「もう変更しないでください」と作家に伝えたいとき、または会期中に内容を凍結したいときに使います。ロック中の作品は、作家側の入力画面では各項目が灰色になり編集できなくなります。

### 個別ロック (作家側からの自主提出)

作家が作品入力画面で **「✓ 提出する」** ボタンを押すと、その 1 件だけがロックされます。「自分はもう編集しません」という意思表示。

### 主催者側で個別ロック / 解除

「作品登録/設定」画面 → **「📋 作品一覧」** タブ → 該当作品を展開 → 編集パネル末尾の **🔒 / 🔓 ボタン**

### 一括ロック / 解除

「⚙️ 項目設定」タブの中段「**作品の一括ロック**」セクションへ:

- **🔒 全作品をロック**: 登録済の全作品を一気にロック (会期直前に推奨)
- **🔓 全作品をロック解除**: 全作品を解除 (会期後の修正受付・トラブル対応時)

> 会期当日は一括ロックしておくと、来場者・SNS 公開後に作家が誤って画像を差し替えてしまう事故を防げます。

## 3.7. 会場・会期中の修正対応

会期が始まってから「タイトルにタイプミスがあった」「価格を変えたい」「画像を撮り直して差し替えたい」といった修正が必要になることがあります。スマホやノート PC から会場でも対応できます。

### A. 主催者が直接修正する (もっとも素早い)

1. スマホかノート PC で「作品登録/設定」画面を開く
2. 主催者ログイン (一度ログイン済みなら数日間維持されます)
3. 「📋 作品一覧」タブで該当作品を探す
4. 行をタップして展開 → 内容を直接編集 → 「💾 更新する」

> 作品がロック中でも主催者には編集制限はかかりません。「ロック中だから直せない」と心配せず編集して大丈夫です。

### B. 作家本人に修正してもらう

作品の差し替えなど大きな修正の場合は、作家自身に直してもらう方が手戻りが少ないことがあります。

1. 該当作品がロック中なら、まず編集パネルから 🔓 解除する
2. 案内メールに記載した作家用 URL を改めて伝える (案内メールを再送してもよい)
3. 作家が入力画面から修正
4. 修正完了の連絡を受けたら、必要に応じて再びロックする

### C. 画像だけ差し替える (会場で撮り直し)

会期前に作品写真が間に合わなかった場合や、撮り直したい場合に便利です。

1. 主催者がスマホで「作品登録/設定」画面を開く
2. 「📋 作品一覧」 → 該当作品の編集パネル
3. 画像欄の「ファイルを選択」 → スマホで撮影、またはアルバムから選択
4. 自動で長辺 1600px に縮小されるので、原寸そのままでも OK
5. 「💾 更新する」

### D. 作家本人が自分のスマホから写真を更新する (ショートカット)

会場で作家本人がいる場合、作家用 URL や入力画面を開かずに、**来場者用 QR からそのまま写真を更新** できる裏口があります。複数作品の写真追加・差し替えを素早く済ませたいときに便利です。

1. 作家が作品の横の **QR コード** を読み取る (来場者用の作品ページが開く)
2. 感想入力フォームの **「ニックネーム」欄に作家名 (登録時と同じ綴り)** を入力
3. 「🎨 作家本人として認識: ○○○○」のメッセージとともに **「📷 写真を選択して更新」** ボタンが現れる
4. 作家が写真を選択 → 自動アップロード

文字情報は変更されず、画像だけ上書きされます。

> ロック中の作品はこの方法でも写真更新できません。事前にロック解除してください。
> 作家には「作品の QR を読んで、ニックネーム欄に自分の作家名を入れると写真が更新できる」と一言伝えれば OK です。詳細は作家向けマニュアル §7.3 にも書いてあります。

### よくある状況

**Q. 値札 (キャプション) を会期途中で刷り直したい**
A. 「キャプション」画面で印刷 → ブラウザの印刷ダイアログで該当ページだけ指定するのが手っ取り早いです。

**Q. 全部の作品を会期中に編集禁止にしたい**
A. §3.5 の「🔒 全作品をロック」を使ってください。誤操作でデータが壊れる事故を防げます。

**Q. ロック中でも来場者のいいね・コメントは普通に集まる?**
A. はい。ロックは作家の編集を止めるだけで、来場者のいいね・コメントには影響しません。

## 4. 感想 (コメント) 受付期間の設定

1. 「⚙️ 項目設定」タブの下方「感想 (コメント) 受付期間」へ
2. 開始/終了の日時を入力 (空欄なら常時受付)
3. 「保存」
4. 来場者の感想画面では、期間外になると入力欄の代わりに「感想の受付期間は終了しました。」(または開始前なら開始日時のお知らせ) が表示されます

## 5. キャプション/QR を印刷する

### 5.1. 画面を開く

1. 「キャプション」画面を展覧会コードを指定して開く
2. 「主催者ログイン」カードが出ている場合は主催者ログイン (§2 と同じ手順)。ログイン済みなら直接展覧会データが読み込まれます

### 5.2. 推奨フロー (4 ステップ)

「🖨️ Caption Print」タブ最上部の「💡 おすすめの使い方」にも書いてありますが、迷ったらこの順序が最短です:

1. **⭐ 公式テンプレを 📥 採用** — 項目設定・フォーム設定・Page タブが一気に揃う
2. **「項目設定」「フォーム設定」タブで微調整** — 必要なら
3. **「Page」タブで余白・QR サイズなど細部調整** — 必要なら
4. **💾 自分のテンプレに保存** — 別テンプレを試す前のセーフネットになる

### 5.3. テンプレ一覧

「🎨 Templates」タブには 9 個のテンプレが並びます。⭐ マーク付きが**公式推奨**:

| | 名前 | 用途 | 採用時に Page 設定も適用 |
|---|---|---|---|
| ⭐ | 🎨 個展 推奨 | 1 人の作家、ゆったり展示 (A7・2×4) | ○ |
| ⭐ | 🎭 グループ展 推奨 | 多人数展、コンパクト (3×5) | ○ |
| ⭐ | 📷 写真展 推奨 | 撮影地・撮影年・カメラ項目 | ○ |
| ⭐ | 🌐 国際展 推奨 | 日英併記、中央寄せ | ○ |
| | ✂️ シンプル | タイトルと作家名のみ | — |
| | 🖼️ ギャラリー | QR 左上 / 画廊風 | — |
| | 🎨 版画・限定版 | エディション・額装価格 | — |
| | 🇯🇵 作品名先 | 和風、作品名から始まる | — |
| | 🇯🇵 作家名先 | 和風、作家名から始まる | — |

**採用時の挙動 (重要):**

| 層 | 挙動 |
|---|---|
| 項目設定 (registration_fields) | **追加** — テンプレが必要とする項目で不足分だけ append、既存項目はそのまま残る |
| フォーム設定 (caption_fields) | **書き換え** — テンプレの並び順・サイズ等で完全置換 |
| Page タブ (cols/rows/QR サイズ/余白 等) | 公式テンプレのみ**書き換え**、それ以外のテンプレは現状維持 |

**カードの見た目:**
- ⭐ + ゴールド枠 = 公式推奨
- 緑グラデ + ✅ 採用中 表記 = いまこの展覧会で採用中のテンプレ
- 「📥 採用」ボタン (薄い青) を押すと採用ダイアログが出ます

### 5.4. 詳細な編集とプリセットへの保存

採用後の細部調整は以下の各タブで行います:

- **項目設定タブ**: 「タイトル」「価格」など各フィールドの有効/無効、必須かどうか
- **フォーム設定タブ**: キャプションに出す項目の並び順、フォントサイズ、太字、最大行数 (1〜3 行・制限なし) など
- **Page タブ**: 用紙サイズ、列数・行数、QR 位置・サイズ、余白、フォントファミリ

満足したら「Template name...」に名前を入れて「💾 Save」で**自分のテンプレ**として保存。
- 同名で保存しようとすると上書き確認ダイアログが出ます
- 「Saved templates」から「Load」で呼び戻し、展覧会またぎで再利用可

### 5.5. QR コードの一括印刷

「🔲 QR Print」タブで QR コードだけを別途一括印刷できます。会場での貼り出しや、QR シールを作品の裏に貼る用途など。

## 5.7. 🌐 Web 展覧会を公開する

会場に来られなかった人向けや、会期後のアーカイブ閲覧用に、作品をオンラインで一覧できる Web 展覧会を公開できます。

### 5.7.1. Web 展覧会の設定画面を開く

主催者専用 URL: `https://rohei-printer-system.web.app/web-exhibition.html?ex=○○○`
(`○○○` は展覧会コード)

主催者メールアドレスでログインしてください。

### 5.7.2. 公開設定

「公開設定」カードで 2 つを選びます。

**公開状態**:
| 選択肢 | 動作 |
|---|---|
| 🔒 **非公開** | 誰も見られません (準備中・休止中はこれ) |
| 🎫 **QR を持つ来場者のみ** | 主催者が発行した会場 QR / リンクを持つ人だけ閲覧可 (会期中の招待制運用) |
| 🌍 **公開** | URL を知っていれば誰でも見られます (SNS 拡散・アーカイブ運用) |

**Web 展覧会で来場者ができること**:
| 選択肢 | 動作 |
|---|---|
| **閲覧のみ** | 作品を見るだけ。いいね・コメントの機能はなし |
| **いいねのみ** | ハートボタンでいいねできる |
| **いいね + コメント** | いいね + 💬 でコメントも書ける |

「💾 設定を保存」で反映されます。

### 5.7.3. ギャラリー外観

「🎨 ギャラリー外観」カードで Web 展覧会の見た目をカスタマイズできます:

- **タイトル / サブタイトル**: SNS でシェアしたときのプレビューにも使われます
- **ヒーロー画像**: ページ上部の大きな画像 + SNS シェア時のプレビュー画像 (公開する展覧会では**設定推奨**)
- **背景画像 / 背景色**: 雰囲気作り (背景画像があればそちらが優先)

「💾 外観を保存」で反映されます。

### 5.7.4. 会場 QR コードの発行 (招待制モード用)

公開状態が「QR を持つ来場者のみ」のときに使う、**有効期限付きの専用 URL** を発行する機能です。会場入口に QR を貼っておけば、来場者がスマホで読み取って Web 展覧会を見られるようになります。期限が切れると無効になるので、会期終了後に意図せずアクセスされる心配がありません。

1. 「🎫 会場 QR コードの発行」カードへ
2. 有効期限 (日数) を入力 (デフォルト 60 日)
3. 「QR を発行」 → URL と QR 画像が表示される
4. ダウンロード → 印刷して会場に貼る、または DM 等で配布

> 公開状態が「公開」モードならこの操作は不要です。Web 展覧会の URL を SNS や Web サイトに貼るだけで誰でも見られます。

### 5.7.5. SNS で告知する

「公開」モードに切り替えた後の SNS 告知方法は **SNS 告知マニュアル** を参照してください。

## 6. 出力・レポート (ハブ画面)

展覧会のリアルタイム表示・データダウンロードへの入口を 1 つにまとめた画面です。「作品登録/設定」画面・「キャプション」画面のヘッダにある **「📊 出力・レポートへ」** から開けます (展覧会コードと認証は引き継がれます)。

ハブには 2 つの行き先があります。

### 📈 リアルタイムダッシュボード

- 参加者数 / いいね数 / コメント数 / 登録作品数の大きな数字
- **いいねランキング 3 種**:
  - 🏆 **総合**: 会場 QR + Web 展覧会 の合算
  - 🏛️ **会場**: 会場 QR からのいいねのみ
  - 🌐 **Web**: Web 展覧会からのいいねのみ
- 最新コメントフィード
- 会場控え室モニタ等に常時表示する用途
- 「→ 開く (別タブ)」をクリックすると新しいタブで開きます

### 📥 データダウンロード

「→ 開く」をクリックすると、展覧会選択済みの状態で開きます。

ダウンロードできる種類:

- 🎨 **Artworks XLSX**: 作品データ。Excel で開いても型崩れしません (エディション「5/30」が日付に化けない・価格や品番の書式も保持されます)
- 💬 **Comments CSV**: コメント全件 (時刻順)
- 👍 **Likes CSV**: いいね一覧
- 📊 **Summary CSV**: 作品別の like/コメント数集計

> ハブ画面・ダウンロード画面ともに、主催者本人 (展覧会作成時に登録したメールアドレス) でログインしないと閲覧できません。展覧会作成時のアドレスとログインアドレスが一致していないと「展覧会がありません」と表示されます。

## 7. トラブルシューティング

- **ログインリンクのメールが届かない**: 迷惑メールフォルダを確認してください。Gmail なら「迷惑メールではない」を一度押すと、以降は受信トレイに届きます。
- **ログインリンクをクリックしたら「期限切れ」「無効」と出る**: リンクには有効期限 (約 1 時間) があります。もう一度メールリンクを送信してください。Outlook 等が先回りでリンクをプレビュー取得してしまうケースもあるので、メールアプリのリンクプレビューを切るか、別のメーラーで開くと回避できます。
- **「展覧会がありません」と表示される (データ書き出し画面)**: ログインしたメールアドレスと、展覧会作成時に登録した主催者メールアドレスが一致していない可能性があります。
- **テンプレートを Load しても項目が出ない**: 現在の展覧会の「項目設定」で当該項目が ON になっていない可能性。項目設定タブで確認してください。
- **画面更新が反映されない**: ブラウザキャッシュが古い可能性。`Ctrl+Shift+R` (Mac: `Cmd+Shift+R`) で強制再読み込みしてください。
- **作品枠を増やしたい**: 「項目設定」タブ末尾「作品枠を追加」
- **過去展覧会の設定を全部使いたい**: 項目設定タブの「📥 他の展覧会から設定を複製」で「項目設定」「キャプション設定」「Web 公開設定」のチェックボックスにすべてチェックを入れて複製してください。キャプションテンプレートの選択もそのまま引き継がれるので、キャプション画面で再選択する必要はありません。
- **作家から「ロックされていて編集できない」と連絡が来た**: 個別解除なら「作品一覧」タブから該当作品を展開して 🔓 ボタン、全件解除なら「項目設定」タブの「🔓 全作品をロック解除」
- **Web 展覧会の URL を SNS に貼ってもプレビュー画像が出ない**: Web 展覧会の設定画面 → ギャラリー外観 で **ヒーロー画像** を設定してください。設定後は SNS 側のキャッシュで古いまま見える場合があるので、SNS 告知マニュアル §4「プレビュー画像の確認と更新」を参照。
- **Web 展覧会で「閲覧には QR コード経由のアクセスが必要です」と表示される**: 公開状態が「QR を持つ来場者のみ」のままです。SNS 拡散したいなら「公開」に切り替えてください。
